# Dummer — Claude Code project brief

## Shell
Always invoke commands via `cmd /c "..."` (Windows, not PowerShell or bash).

## Project snapshot
ESP32 drum machine — PCM5102 DAC, I2S, 7 buttons, 5 LEDs.
Source of truth: `SPEC.md`. Read it before any non-trivial feature.
Current branch: `main`. Milestones M1–M7 complete.

## Build
```
scripts\test.bat                   # host unit tests
scripts\build_all.bat              # full pipeline (codegen + all envs + tests)
```
Every code change must end with a green `build_all.bat`.

## Code rules
- **No heap allocation** in production audio/control code (`new`, `malloc`, `std::vector`, etc. are forbidden outside of setup-time statics).
- All hardware access through HAL interfaces only — modules in `audio/`, `control/`, `app/` must compile clean on `native_test`.
- Audio arithmetic is **Q15 fixed-point** (`int32_t` accumulators, saturate-clip to `int16_t`).
- Super-loop: nothing in the main task may block for more than ~1 ms.
- No comments unless the WHY is non-obvious. No docstrings.
- C++17, `-Wall -Wextra -Wshadow -Wold-style-cast -Wcast-align -Wnon-virtual-dtor`.

## Key defines (see `include/defines.hpp`)
`AUDIO_CHUNK=64`, `SAMPLE_RATE=44100`, `MAX_VOICES=50`, `LOOPER_MAX_EVENTS`, `LOOPER_MAX_DURATION_S`.

## Audio quirks
- PCM5102 needs **offset binary** (XOR sign bit into upper 16 bits of 32-bit slot).
- Retrigger fade: `gain *= RETRIGGER_FADE_K_Q15` (≈ −60 dB in 23 ms).
- Universal end ramp: 64-sample linear fade on natural sample end.
- Reverb off by default; btn3 toggles; red LED reflects state.

## Testing pattern

Add new test environments to `platformio.ini` `test_filter`.

## MIDI / auto-drummer (M6)
Pattern files live in `assets/patterns/` (SMF format-0, PPQN=480).
Generated by `scripts/gen_patterns.py`.
LOOP_START marker at tick 1920 (bar 1) — bar 0 is the stick count-in; loop runs from bar 1.
`scripts/midi_to_cpp.py` converts them to `generated/patterns.hpp/cpp`.

## Velocity & humanization (M7)
- MIDI velocity is **live**: `AutoDrummer::tick()` emits a parallel `vel[]`; `VoicePool::trigger(sid, vel)` scales `gain = MASTER_VOLUME_Q15 * vel / 127`. So velocity = per-hit musical dynamics.
- Per-instrument **loudness** is the PCM gain in `scripts/wav_to_cpp.py` (e.g. crash 0.2, ride 0.35), NOT velocity. Crash/ride pattern velocities are pinned to 127 so PCM stays the loudness authority; routing velocity does not double-attenuate them.
- `Humanizer` (`audio/humanizer.hpp`) is a 16-bit Galois LFSR. Timing jitter ±`HUMANIZE_TIME_SAMPLES` (~±5 ms) is re-drawn per loop iteration into `*_play_[]` with a monotonic + in-range clamp (scan never reorders/drops). Velocity jitter ±`HUMANIZE_VEL` applied at fire time. Toggle via `AutoDrummer::set_humanize()`; on by default (`HUMANIZE_ENABLED_DEFAULT`).
